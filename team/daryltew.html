PROJECT: WorkoutBook

Overview

Are you a professional athlete who is too lazy to manually search through your workout list to find your favourite workout? Are you an amateur who just started working out and doesnâ€™t know where to begin? Or are you just someone who prefers to have all of your workouts in one place and organized in a tidy manner. Then WorkoutBook is the solution for you! WorkoutBook is a desktop workout book application used for anyone who does workouts to manage their workouts. The user interacts with it using a CLI, and it has a GUI created with JavaFX.

Summary of contributions

  • Major enhancement: added the memory feature which enables users to manage future, current and completed workouts

    • What it does: This allows the WorkoutBook user add new workouts with an state tag of future being attached automatically. In order to set a workout to current, he uses the current command which changes the state tag to current. He can then finish the workout by setting the state tag to completed using the complete command. However, he can always choose to reactivate the workout by changing its state tag to current using the current command again. This flexibility enables users to fit the workouts to their changing preferences.

    • Justification: Users can now remember easily the relevance of each workout according to their desired criteria. future reminds the user that they are untouched. current informs the user that he is focussing on this specific one. completed tells the user that he has finished this workout before. This is advantageous to the user who can now avoid forgetting whether he has already performed the workout or not.

    • Highlights: This feature requires two new commands - current and complete as well as modification to the pre-existing add and edit commands to avoid affecting the state tag.

Changing a workout to completed : complete

Changes a workout to be a completed workout in the workout book.
Format: complete INDEX

  • Changes the state tag of the workout at the specified INDEX. The index refers to the index number shown in the displayed workout list. The index must be a positive integer 1, 2, 3, …​

  • You can change the state tag only from current.

Examples:

  • complete 1
    Changes the state tag of the 1st workout to be completed.

Current Implementation

The memory mechanism is facilitated by VersionedAddressBook. It extends WorkoutBook. It implements following commands:

  • CurrentCommand() - Changes the state tag of a workout by index number to current

  • CompleteCommand() - Changes the state tag of a workout by index number to completed

Given below is an example usage scenario and how the memory mechanism behaves at each step.

Step 1. The user launches the application, add some workouts to the workout book and they automatically have a future tag.

Step 2. The user executes the current command to change the future tag in the workout as specified by the index number to current.

Figure 1. Interactions inside Logic component when pressing current

Step 3. The user executes the complete command to change the current tag in the workout as specified by the index number to completed.

Figure 2. Interactions inside Logic component when pressing complete

  • The user can only change the state tag from current to completed by using the complete command.

  • The user can only change the state tag from future to current or completed to current by using the current command.
